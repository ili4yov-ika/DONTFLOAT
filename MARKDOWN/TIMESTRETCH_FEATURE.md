# Функция растяжения времени (Time Stretch)

**Дата создания**: 2025-01-08  
**Обновлено**: 2026-05-31  
**Версия проекта**: 0.0.0.1  
**Статус**: ✅ Реализовано (метки, Ctrl+T, undo/redo, Rubber Band R3)

## Обзор

Функция растяжения времени позволяет изменять длительность аудиофрагментов без изменения высоты тона (pitch). Реализована с использованием меток цикла A и B для определения области трансформации.

## Архитектурные компоненты

### MarkerEngine

**Файлы**: `include/markerengine.h`, `src/markerengine.cpp`

**Роль**: Данные и UI меток растяжения (логика растяжения перенесена в TimeStretchProcessor)

**Ответственность**:
- Структуры **MarkerData** (данные: позиция в сэмплах, флаг isFixed) и **Marker** (UI: отображение, перетаскивание)
- Валидация меток: нулевая метка статична, минимальный сегмент 50 мс, без дубликатов, метки не перескакивают
- Сортировка по позиции, ограничения при перетаскивании

**Основные методы**:
- Конструкторы и конвертация MarkerData ↔ Marker
- Валидация и расчёт сегментов — в TimeStretchProcessor

### TimeStretchProcessor

**Файлы**: `include/timestretchprocessor.h`, `src/timestretchprocessor.cpp`

**Роль**: Обработчик временного растяжения аудио

**Ответственность**:
- Алгоритмы растяжения/сжатия времени
- Поддержка тонкомпенсации (pitch correction)
- Сохранение качества аудио при трансформации
- Работа с аудиобуферами
- Интеграция с системой команд (undo/redo)

**Основные методы**:
- `applyMarkerStretch()` — применить растяжение по списку меток (MarkerData)
- `calculateSegments()`, `validateMarkers()`, `calculateStretchFactor()` — подготовка и валидация
- `processWithPitchPreservation()` — тонкомпенсация через `RubberBandOffline::stretchMono()`
- Результат в структуре `StretchResult`

### RubberBandOffline

**Файлы**: `include/rubberband_offline.h`, `src/rubberband_offline.cpp`, `cmake/RubberBand.cmake`

**Роль**: Офлайн time-stretch одного канала (обёртка над Rubber Band Library v4)

**Ответственность**:
- Движок **R3** (`OptionEngineFiner`), офлайн-режим (`study` + `process`)
- Статическая библиотека `rubberband_single` из `thirdparty/rubberband/single/RubberBandSingle.cpp`
- Лицензия: GPL-2.0-or-later (согласована с DONTFLOAT, GPLv3)

**Основной метод**:
- `stretchMono(input, timeRatio, sampleRate)` — растяжение/сжатие моно-буфера с сохранением высоты тона

### TimeUtils

**Файлы**: `include/timeutils.h`, `src/timeutils.cpp`

**Роль**: Утилиты для работы со временем

**Ответственность**:
- Конвертация между форматами времени (сэмплы ↔ секунды ↔ минуты:секунды)
- Вычисления временных интервалов
- Форматирование временных меток для отображения
- Преобразование между позициями воспроизведения и временем

**Основные методы**:
- `samplesToTime(qint64 samples, int sampleRate)` - конвертация сэмплов во время
- `timeToSamples(double time, int sampleRate)` - конвертация времени в сэмплы
- `formatTime(double seconds)` - форматирование времени для отображения
- `parseTime(const QString& timeString)` - парсинг строки времени

## Пользовательский интерфейс

### Установка меток цикла

**Горячие клавиши**:
- `A` - установить метку A (начало области)
- `B` - установить метку B (конец области)
- `Shift+A` - удалить метку A
- `Shift+B` - удалить метку B

**Визуальное отображение**:
- Метки A и B отображаются на WaveformView вертикальными линиями
- Область между метками подсвечивается полупрозрачным цветом
- Текущие позиции меток отображаются в полях ввода

### Применение трансформации

**Горячая клавиша**: `Ctrl+T` - применить сжатие-растяжение по меткам

**Процесс**:
1. Пользователь устанавливает метки A и B
2. Выбирает целевой BPM или коэффициент растяжения
3. Нажимает `Ctrl+T` для применения трансформации
4. Появляется диалог прогресса с возможностью отмены
5. После завершения аудио обновляется с возможностью отмены (Ctrl+Z)

**Параметры**:
- **Целевой BPM**: Новый темп трека (автоматически рассчитывается коэффициент)
- **Коэффициент растяжения**: Прямое указание коэффициента (1.0 = без изменений, >1.0 = растяжение, <1.0 = сжатие)
- **Тонкомпенсация**: Включить/выключить сохранение высоты тона
- **Качество**: Быстрое/Нормальное/Высокое

## Алгоритмы обработки

### Phase Vocoder (Фазовый вокодер)

**Статус**: Планируется к реализации

**Описание**: Классический алгоритм временного растяжения с сохранением высоты тона

**Преимущества**:
- Хорошее качество для музыкальных треков
- Стабильная работа с различными типами аудио
- Широко используется в профессиональных приложениях

**Недостатки**:
- Может создавать артефакты при больших коэффициентах растяжения
- Требует значительных вычислительных ресурсов

### Rubber Band Library (R3 offline)

**Статус**: ✅ Реализован (2026-05-31) в `RubberBandOffline::stretchMono()`, вызывается из `TimeStretchProcessor::processWithPitchPreservation()`

**Описание**: Библиотека временного растяжения с сохранением высоты тона; в проекте — single-file сборка v4, движок R3 (`OptionEngineFiner`), офлайн-обработка сегментов

**Преимущества**:
- Качество на уровне профессиональных DAW/DJ-приложений (Mixxx, Sonic Visualiser)
- Стабильное сохранение pitch при коэффициентах ~0.5–2.0
- Проверенная GPL-библиотека с активной поддержкой

**Недостатки**:
- Офлайн-обработка (не в реальном времени)
- Дополнительная зависимость (`thirdparty/rubberband`, GPL-2.0+)
- Больше вычислительных затрат, чем у простого WSOLA

**Зависимость**:
```bash
git clone --depth 1 --branch v4.0.0 https://github.com/breakfastquay/rubberband.git thirdparty/rubberband
```

Подробнее: [thirdparty/README.md](../thirdparty/README.md)

### WSOLA (исторически)

**Статус**: Заменён Rubber Band (2026-05-31)

## Интеграция с системой команд

### TimeStretchCommand

**Файлы**: `include/timestretchcommand.h`, `src/timestretchcommand.cpp`  
**Статус**: ✅ Реализовано

**Описание**: Команда для отмены/повтора операций растяжения по меткам (Ctrl+T). Регистрируется в QUndoStack MainWindow.

**Наследование**: `QUndoCommand`

**Данные команды**:
- Исходные аудиоданные и метки (для отмены)
- Обработанные аудиоданные и метки (для повтора)
- Ссылка на WaveformView для обновления отображения

**Методы**:
- `undo()` - восстановление исходных данных
- `redo()` - повторное применение трансформации
- `id()` - идентификатор команды для объединения

**Оптимизация памяти**:
- Хранение только измененной области (между метками A и B)
- Сжатие неиспользуемых данных
- Ограничение глубины истории отмены

### Рекомендации по использованию

**Для лучшей производительности**:
- Используйте быстрое качество для предпросмотра
- Ограничьте область трансформации (используйте метки A и B)
- Отключите тонкомпенсацию, если не требуется сохранение высоты тона

**Для лучшего качества**:
- Используйте высокое качество для финальной обработки
- Избегайте экстремальных коэффициентов растяжения (рекомендуется 0.5-2.0)
- Включайте тонкомпенсацию для музыкальных треков

## Ограничения и известные проблемы

### Текущие ограничения

1. **Размер обрабатываемого фрагмента**: Для больших областей может потребоваться значительное количество памяти
2. **Экстремальные коэффициенты**: Коэффициенты < 0.25 или > 4.0 могут давать неудовлетворительное качество
3. **Реальное время**: Полная обработка не происходит в реальном времени, требуется ожидание завершения; для коротких фрагментов доступен облегчённый предпросмотр при перетаскивании меток (без тонкомпенсации)

### Известные проблемы

1. **Артефакты при больших изменениях**: При коэффициентах > 2.0 или < 0.5 могут появляться слышимые артефакты
2. **Производительность**: Обработка больших файлов (> 5 минут) может занимать заметное время
3. **Память**: При обработке очень больших файлов может потребоваться много RAM

## Примеры использования

### Базовое использование через UI

```
1. Загрузить аудиофайл (Ctrl+O)
2. Установить метку A в начале области (клавиша A)
3. Установить метку B в конце области (клавиша B)
4. Нажать Ctrl+T для применения трансформации
5. В диалоге выбрать параметры (BPM, тонкомпенсация, качество)
6. Подтвердить и дождаться завершения обработки
7. При необходимости отменить изменения (Ctrl+Z)
```

### Программное использование

```cpp
// Метки (MarkerData) из WaveformView, процессор
TimeStretchProcessor processor;
QVector<MarkerData> markers = ...; // из view
StretchResult result = processor.applyMarkerStretch(originalSamples, sampleRate, markers);

// Интеграция с системой команд (в MainWindow)
TimeStretchCommand* command = new TimeStretchCommand(view, view->getMarkerData(), result.processedData, ...);
undoStack->push(command);
```
