### 2. Визуализация

#### Статус: ✅ АКТУАЛИЗИРОВАНО (обновлено 2026-01-25)
**Описание**: Упрощён и уточнён набор визуализаций в `WaveformView`.

**Текущее**:
- ✅ **Визуализация отклонений битов** — включена при `showBeatDeviations` и наличии битов. Вызов `BeatVisualizer::drawBeatDeviations()` из `WaveformView::paintEvent()` в цикле по каналам с клипом по области канала.
  - **Геометрия**: Ровно 3 вершины — основание (вертикаль) на **фактической** позиции бита (`actualX`), вершина на **ожидаемой** (`expectedX`) в центре по Y. Одна точка вершины `apexY = rect.center().y()` для обеих диагоналей, чтобы форма была треугольником, а не параллелограммом. Путь: `QPainterPath` с `Qt::WindingFill`, координаты через `QPointF`/`qreal`.
  - **Ожидаемая позиция**: используется `beat.expectedPosition` из анализатора при `expectedPosition != position`, иначе расчёт от предыдущего бита + интервал по BPM.
  - **Направление**: треугольник указывает от фактической позиции к ожидаемой (стрелка к сетке).
  - **Цвета**:
    - Растянутые (deviation > 0): красный (яркий / бледный в зависимости от `beatsAligned`)
    - Сжатые (deviation < 0): синий (яркий / бледный)
  - **Порог**: 2% по умолчанию (`deviationThreshold = 0.02f`); биты с `|deviation| < threshold` не рисуются.
  - **Отрисовка**: сплошная заливка треугольника + полупрозрачный фильтр (alpha=30) по клипу пути + контур. Антиалиасинг включён для заливки. Вертикальная линия на фактической позиции бита.
  - **Клип**: в виджете волны перед вызовом `drawBeatDeviations` устанавливается `painter.setClipRect(channelRect)`, после — `restore()`.
  - **✅ МИГРИРОВАНО (2026-01-12)**: Код в `BeatVisualizer`; вызов из `WaveformView`.
- ✅ Силуэт ударных поверх волны (опция) — `BeatVisualizer::drawBeatWaveform()`
- ⚠️ **Известная проблема**: Механизм поиска неровных долей может работать некорректно или давать неверные результаты (см. раздел "Известные баги" → "5. Механизм поиска неровных долей")

**Реализация**:
- `src/beatvisualizer.cpp` — статические методы `drawBeatDeviations()`, `drawBeatWaveform()`, `createDiagonalPattern()`
- `include/beatvisualizer.h` — структуры `VisualizationSettings`, `BeatDeviationColors`, объявления методов
- `src/waveformview.cpp` — делегирование отрисовки в `BeatVisualizer`, методы-обёртки для управления настройками
- `include/waveformview.h` — хранение настроек `beatVisualizationSettings` и `beatDeviationColors`
- `src/mainwindow.cpp` — переключатель "Пометить неровные доли", установка флага выравнивания
- `src/beatfixcommand.cpp` — управление флагом выравнивания при undo/redo

**Заметки по UX**:
- Визуализация использует полигоны с диагональными полосками вместо сплошной заливки для лучшей читаемости
- **Треугольная форма** области отклонений (стрелка вправо) наглядно показывает направление от ожидаемой к фактической позиции
- Разные цвета для растянутых (красный) и сжатых (синий) долей улучшают понимание типа отклонения
- Бледные цвета для невыровненных долей помогают отличить их от выровненных
- Противоположные направления диагоналей для растянутых (вправо `\`) и сжатых (влево `/`) долей создают визуальное различие
- Полоски сделаны тонкими (4px ширина) с небольшим расстоянием (3px) для большего количества и лучшей видимости волны
- Добавлен бледный фильтр поверх полигонов для улучшения видимости звуковой волны

**Технические детали реализации (2026-01-25)**:
- ✅ **Миграция кода**: Визуализация ударных в `BeatVisualizer`; вызов из `WaveformView::paintEvent()` при `showBeatDeviations && !beats.isEmpty()`.
- ✅ **Геометрия треугольников**: Основание — вертикальный отрезок на `actualX`; вершина — одна точка `(expectedX, rect.center().y())`. Обе диагонали сходятся в этой вершине (`QPainterPath`: moveTo(base, top) → lineTo(apex) → lineTo(base, bottom) → close). Используются `QPointF`, `qreal`, `setFillRule(Qt::WindingFill)`.
- ✅ **Ожидаемая позиция**: Из анализатора `beat.expectedPosition` (если ≠ `position`), иначе `beats[i-1].position + expectedBeatInterval`.
- Заливка — сплошной цвет (`baseColor` + `deviationAlpha`), поверх — клип по пути и полупрозрачный фильтр, затем контур. Метод `createDiagonalPattern()` есть в коде, но для треугольников сейчас используется только сплошная заливка.
- Доп. план по развитию визуализации: [PLAN_BEAT_DEVIATION_NEXT_STEPS.md](PLAN_BEAT_DEVIATION_NEXT_STEPS.md).

**Известные проблемы**:
- ⚠️ **Визуализация треугольников отклонений**: Внесены правки (2026-01-25): единая вершина по `rect.center().y()`, явные `QPointF`, база по фактической позиции, вершина по ожидаемой. Если форма по-прежнему отображается неверно: выполнить полную пересборку (Clean + Rebuild); убедиться, что включён показ отклонений и что у части битов `|deviation| ≥ deviationThreshold` (по умолчанию 2%). Жёлто-оранжевый силуэт — это `drawBeatWaveform()` (силуэт ударных), а не треугольники отклонений (красный/синий).
- ⚠️ **Механизм поиска неровных долей**: Алгоритм определения отклонений может работать некорректно (см. раздел "Известные баги" → "5. Механизм поиска неровных долей").

**Известные проблемы производительности**:
- ⚠️ **Силуэт ударных**: При отображении силуэта ударных (`drawBeatWaveform()`) наблюдается заметное снижение производительности, особенно на больших аудиофайлах. Это связано с вычислением энергетической огибающей для каждого бита и отрисовкой полигонов. Рекомендуется отключать силуэт ударных при работе с длинными треками или при необходимости плавной прокрутки.

**Причина**:
- Реализация `processWithPitchPreservation()` фактически делала ресемплинг окон и не сохраняла высоту тона
- Визуально метки работали корректно, но звук вел себя как при обычном изменении скорости

**Решение**:
- Переписан алгоритм тонкомпенсации на WSOLA (Waveform Similarity Overlap-Add)
- Используется корреляционный поиск в окнах и перекрытие с нормализацией
- Результат сохраняет тембр/высоту тона при изменении длительности

**Файлы**:
- `src/timestretchprocessor.cpp` — новый алгоритм WSOLA в `processWithPitchPreservation()`

**Правила системы меток (2025-02-02)**:
1. **Нулевая метка статична** — начальная метка всегда в 0:00, `isFixed=true`, не перетаскивается; после сортировки и в `setMarkers()` принудительно `position = 0`.
2. **Нет дубликатов** — нельзя создать несколько меток в одном месте; при добавлении проверка `qAbs(markers[i].position - position) < minSegmentSamples`; при перетаскивании запрет совпадения с другими метками.
3. **Метки не перескакивают** — при перетаскивании позиция ограничена соседними метками; сортировка по позиции после каждого изменения.
4. **Минимальный сегмент 50 мс** — константа `MIN_MARKER_SEGMENT_MS = 50`; новый сегмент между двумя метками не может быть меньше 50 мс; в сегменте длиной 50 мс нельзя создавать новые метки (между соседями должно быть ≥ 100 мс для вставки); при перетаскивании ограничение `prev + minSegSamples` и `next - minSegSamples`.
5. **Быстрая отрисовка при перемещении** — при перетаскивании метки вызывается `update()` (мгновенная перерисовка), без throttling.
